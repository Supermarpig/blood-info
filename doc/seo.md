## 三、根本原因分析

### 問題 1：域名沒有權威性

- ~~目前使用 `blood-info.vercel.app`（Vercel 子域名）~~ → **已換為 `www.bloodtw.com`（2026-03-27）**
- 沒有累積的反向連結（backlinks）
- 相比官方 `donate.blood.org.tw` 毫無競爭力

### 問題 2：首頁可見文字內容太少

- `app/page.tsx` 的 H1 只有「捐血活動查詢」5 個字
- 主要內容是動態載入的活動列表（JavaScript 渲染）
- Google 爬蟲看到的有效文字非常有限

### 問題 3：「今天哪裡有捐血車」標題沒對應到

- 這個查詢有 21 次曝光但 0 點擊
- 首頁 title/description 沒有直接包含這類問句型關鍵字
- 用戶看到搜尋結果沒有被吸引點擊

### 問題 4：地區頁缺少固定文字介紹

- `/region/[slug]` 頁面只有活動列表
- 缺少每個縣市的固定介紹段落（Google 喜歡有實質內容的頁面）

### 問題 5：長尾關鍵字頁面不足

- 沒有針對「今天哪裡有捐血車」這類問句型查詢的專屬頁面
- 缺少年份頁面（如「2026 捐血贈品」）

- **狀態：✅ 完成（2026-03-27）**
- **目標：** 新增台北市、新北市、桃園市、台中市、台南市、高雄市、基隆市等城市頁
- **原因：** 目前只有 4 個大區頁（北區/桃竹苗/中區/南區），但使用者搜尋的是「台北捐血」「新北捐血」等城市精確關鍵字

#### ✅ 2. 各動態頁面補上 `canonical` tag

- **狀態：✅ 完成（已存在）**
- **目標：** 在 `generateMetadata()` 的 region、gift 頁加入 `alternates.canonical`
- **原因：** 目前 canonical 只在根 layout 設定，各子頁面缺乏明確 canonical，Google 可能不確定哪個 URL 是權威版本
- **預期效果：** 避免重複內容問題，強化各頁面的索引正確性

### 🟡 中優先

#### ✅ 3. JSON-LD 加入 `dateModified`

- **狀態：✅ 完成（2026-03-27）**

- **原因：** Google 用 dateModified 判斷內容新鮮度，有助於在「今日」類查詢中排名更前
- **預期效果：** 爬蟲更快重新索引更新後的活動資料

### AI 爬蟲 vs 傳統 SEO 差異

| 面向         | 傳統 Google SEO        | AI 搜尋（GEO）                   |
| ------------ | ---------------------- | -------------------------------- |
| 排名邏輯     | 反向連結 + 關鍵字密度  | 被其他來源引用、內容可信度       |
| 最佳格式     | 標題層次 + meta 標籤   | 直接回答問題的段落、FAQ 格式     |
| 更新頻率影響 | Google 重新爬取才更新  | 部分 AI 有即時搜尋能力（Perplexity） |
| 關鍵檔案     | sitemap.xml            | llms.txt（新標準）               |
