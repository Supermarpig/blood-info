## 三、根本原因分析

### 問題 1：域名沒有權威性

- ~~目前使用 `blood-info.vercel.app`（Vercel 子域名）~~ → **已換為 `www.bloodtw.com`（2026-03-27）**
- 沒有累積的反向連結（backlinks）
- 相比官方 `donate.blood.org.tw` 毫無競爭力

### 問題 2：首頁可見文字內容太少

- `app/page.tsx` 的 H1 只有「捐血活動查詢」5 個字
- 主要內容是動態載入的活動列表（JavaScript 渲染）
- Google 爬蟲看到的有效文字非常有限

### 問題 3：「今天哪裡有捐血車」標題沒對應到

- 這個查詢有 21 次曝光但 0 點擊
- 首頁 title/description 沒有直接包含這類問句型關鍵字
- 用戶看到搜尋結果沒有被吸引點擊

### 問題 4：地區頁缺少固定文字介紹

- `/region/[slug]` 頁面只有活動列表
- 缺少每個縣市的固定介紹段落（Google 喜歡有實質內容的頁面）

### 問題 5：長尾關鍵字頁面不足

- 沒有針對「今天哪裡有捐血車」這類問句型查詢的專屬頁面
- 缺少年份頁面（如「2026 捐血贈品」）

## 八、站內 SEO 待辦清單（2026-03-27 新增）

### 🔴 高優先

#### ✅ 1. 城市層級頁面 `/city/[slug]`

- **狀態：✅ 完成（2026-03-27）**
- **目標：** 新增台北市、新北市、桃園市、台中市、台南市、高雄市、基隆市等城市頁
- **原因：** 目前只有 4 個大區頁（北區/桃竹苗/中區/南區），但使用者搜尋的是「台北捐血」「新北捐血」等城市精確關鍵字
- **預期效果：** 一次新增 6-8 個獨立 URL，每個可獨立排名，捕捉大量城市長尾流量

#### ✅ 2. 各動態頁面補上 `canonical` tag

- **狀態：✅ 完成（已存在）**
- **目標：** 在 `generateMetadata()` 的 region、gift 頁加入 `alternates.canonical`
- **原因：** 目前 canonical 只在根 layout 設定，各子頁面缺乏明確 canonical，Google 可能不確定哪個 URL 是權威版本
- **預期效果：** 避免重複內容問題，強化各頁面的索引正確性

---

### 🟡 中優先

#### ✅ 3. JSON-LD 加入 `dateModified`

- **狀態：✅ 完成（2026-03-27）**
- **目標：** 在 Event schema、CollectionPage schema 加上 `dateModified` 欄位（對應 ISR revalidate 時間）
- **原因：** Google 用 dateModified 判斷內容新鮮度，有助於在「今日」類查詢中排名更前
- **預期效果：** 爬蟲更快重新索引更新後的活動資料

#### ✅ 4. 各地區頁加入地區專屬 FAQ（含 FAQPage JSON-LD）

- **狀態：✅ 完成（2026-03-27）**
- **目標：** 在 `/region/[slug]` 頁底部加入 3-5 個地區專屬 Q&A，並附 FAQPage schema
- **範例問題：**
  - 北區：「台北哪裡可以捐血？」「新北捐血車時間表怎麼查？」
  - 南區：「高雄哪裡有捐血中心？」
- **預期效果：** 地區問句型關鍵字出現在搜尋結果的 FAQ rich result

#### ✅ 5. `HowTo` Schema（捐血流程）

- **狀態：✅ 完成（2026-03-27）**
- **目標：** 在 FAQ 頁或首頁加入 HowTo schema，說明捐血流程步驟
- **原因：** 搜尋「怎麼捐血」「捐血流程」的使用者是高意圖族群，HowTo rich result 在搜尋頁佔版面大
- **預期效果：** 在「捐血流程」類查詢出現步驟列表 rich result
